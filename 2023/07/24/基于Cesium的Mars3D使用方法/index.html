基于Cesium的Mars3D使用方法

基于Cesium的Mars3D使用方法

最近在做相关的项目，就记录一下自己对轻量化的数据平台Mars3D和轻量化服务器Nginx的理解。

一、基础资源列表

  1. Nginx 站点的全部代码内容
  2. mapdata 用于存放平台使用的本地地理数据，包括矢量、地形、影像、模型等等。可以根据自己的需要在里面进一步细分存储空间。

Nginx下的各文件夹的作用：

conf 服务器配置文件夹

contrib 扩展文件夹

docs 文档文件夹

html 网页界面相关文件夹

logs 日志文件夹

temp 临时文件夹

二、主要的配置文件

1. “nginx-service.xml” 文件

服务器部署文件，在里面修改网站的部署地址。

2. “conf\host\localhsot_80.conf” 文件

本地部署时的端口和数据路径设置。

修改本地地图数据的路径。

3. “conf\nginx.conf” 文件

服务器性能调用配置文件，调用性能与硬件性能匹配以达到最佳效果。

4. “html\index.html” 文件

网站起始页码，内容比较简洁，大部分内容都放在外置的js文件里了。

5. “html\js\index.js” 文件

网站起始页的代码，包括了初始化和开场动画，以及不同设备的优化。
旁边的server.js还包含了用户访问、登录方面的设置，但目前好像并没有用上。

6. “html\config\config.json” 文件

用来导入数据的主要配置文件。因为内容较多会显得比较冗长杂乱，后期可以通过代码的完善。

  • “scene”
    基础场景对象，加载的时候会初始化一个center对象，用来改变镜头的位置和角度。

  • “center”
    lat 纬度 ；lng 经度 ； alt 高程 ；heading 镜头朝向的方位角 ； pitch 俯仰角 ；roll 翻滚角。

  • “terrain”
    地形对象，可以通过url调用地形数据，在三维场景中制造地形效果。

  • “basemaps”
    底图对象，通过url调用基础地图和基础影像服务。

  • “layers”
    图层对象，用来导入各种矢量、影像、模型等数据。矢量需要转成geojson格式，模型也需要转成浏览器支持的gltf、3Dtiles格式。

三、几种数据的加载方式

  1. “geojson”格式 {
         "id": 203013,
     "pid": 2030,
     "type": "3dtiles",
     "name": "村庄",
     "url": "//Kwindygtx/h/mapdata/3dtiles/村庄模型157/Data.scene.json",
     "maximumScreenSpaceError": 2,
     "maximumMemoryUsage": 2048,
     "dynamicScreenSpaceError": true,
     "cullWithChildrenBounds": false,
     "skipLevelOfDetail": true,
     "preferLeaves": true,
     "position": { "alt": 54.41342228 }, 【高程】
     "center": { "lat": 18.63970149, "lng": 109.9013823, "alt": 200, "heading": 3.1, "pitch": -22.9, "roll": 0 }【摄像头位置】
 },
    start nginx nginx.exe -s reload nginx.exe -s stop \快速停止nginx
    nginx.exe -s quit \完整有序的停止nginx location / {
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods “GET, POST, OPTIONS”;
    add_header Access-Control-Allow-Headers “Authorization, Origin, X-Requested-With, Content-Type, Accept”;
    }
    ```
    上述配置中：
  • Access-Control-Allow-Origin * 允许来自任意域的请求。
  • Access-Control-Allow-Methods 指定允许的HTTP方法。
  • Access-Control-Allow-Headers 指定允许的请求头。

1.保存配置文件并重新加载Nginx服务，使配置生效。
这样配置后，Nginx服务器将在响应中添加相应的CORS头，允许跨域请求访问资源。
请注意，配置中的 * 可以指定允许的具体域名，或使用通配符指定允许的域名模式。根据你的实际需求，可以根据需要进行配置。
另外，如果你的Nginx服务器后面有反向代理或负载均衡器，你可能还需要在代理服务器上进行相应的CORS配置，以确保跨域请求能够正确传递到Nginx服务器。
