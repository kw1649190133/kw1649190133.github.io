RTMP流媒体服务器搭建

理论学习

基础概念

RTMP协议

RTMP协议是Real Time Message Protocol(实时信息传输协议)的缩写，它是由Adobe公司提出的一种应用层的协议，用来解决多媒体数据传输流的多路复用（Multiplexing）和分包（packetizing）的问题。该协议基于 TCP，是一个协议族，包括 RTMP 基本协议及 RTMPT/RTMPS/RTMPE 等多种变种。

流媒体

流媒体 (streaming media) 是指将一连串的媒体数据压缩后，经过网络分段发送数据，在网络上即时传输影音以供观赏的一种技术与过程，此技术使得数据包得以像流水一样发送；如果不使用此技术，就必须在使用前下载整个媒体文件。

推流与拉流

推流是将本地的音视频流推送到流媒体服务器上
拉流是拉流是从流媒体服务器上接收音视频流并在本地播放或处理的过程。

推拉流工具

FFmpeg是一套可以用来记录、转换数字音频、视频，并能将其转化为流的开源计算机程序。采用LGPL或GPL许可证。它提供了录制、转换以及流化音视频的完整解决方案。它包含了非常先进的音频/视频编解码库libavcodec，为了保证高可移植性和编解码质量，libavcodec里很多code都是从头开发的。

流媒体的运行流程

通过FFmpeg等工具推送到服务器上保存，然后用户端再通过访问服务器来获取流媒体画面，或者直接下载文件。

ffmpeg推流测试

推流服务器地址格式：
rtmp://+ip+”:”端口+/直播间名字/+”数据流名称” //数据流是我们任意起的名字，用于区分不同的数据来源。
ip地址由服务器决定。
端口和直播名称由配置文件nginx.conf设置，这里的端口是rtmp服务的端口。端口默认为1935，需要注意去服务器防火墙管理开启相应的端口。
直播间名字就是配置文件rtmp{server{application后面的参数名称，而我的配置文件中写的是mylive，所以这里是mylive。
例如：
rtmp://1.14.204.238:1935/mylive/test

推送USB摄像头画面

让AI写了推送摄像头画面的代码，直接放到ubantu里面运行。居然成功了。效果还可以，延时和之前一样大概10S左右。
这下可以吹牛逼说可以给无人机做远程推流了，只要无人机有linxu系统，并且联网，再接个摄像头，再搭建一下服务器，就可以从后台远程查看无人机的画面了。就是延时有点大，这一点和图传还是不一样的。

参数解析：

-f v4l2: 指定输入格式为Video4Linux2，通常用于Linux系统中的摄像头。
-framerate 30: 设置帧率为30帧/秒。
-video_size 1280x720: 设置视频分辨率为1280像素宽和720像素高。
-i /dev/video0: 指定输入设备文件路径。取决于摄像头在系统中的编号。
-f flv: 输出格式设置为FLV，FLV是一种流媒体格式，常用于RTMP协议传输。
rtmp://server/live/streamKey: 替换为你的RTMP服务器地址和流的键。这里server需要带端口，例如1.14.204.238：1935

其中摄像头编号可以用v4l2-ctl --list-devices指令来查看其他的都是比较固定的参数了。

使用其他工具进行推拉流测试

OBS推流测试

【OBS studio网站】https://obsproject.com/zh-cn
这是一个开源的轻量化直播平台，可以比较方便的进行实时推流的测试。
打开OBS之后，要修改直播推流的地址。选择自定义服务，服务器填的是和上面一样的推流服务器地址：rtmp://+ip+":"端口+/直播间名字
而推流码就是类似ffmpeg的推流”数据流名称”，可以自己定义，获取画面的时候要用到。
除此之外还需要设置数据来源，比如将当前显示器画面作为推送画面。设置好之后点击开始直播就可以推送直播画面了。
这算是顺便弄懂了直播是怎么玩的了，所以经常看到游戏主播们开一台电脑作为直播推流机器，另一台专门主机用来专门打游戏，输出画面，这种分布的方式可以避免游戏机器高负载之后把直播卡掉的情况。还有就是用多个画面和图片来组合自己的直播推流画面，做出精美的效果。这还算是复杂的，如果是唱歌跳舞话疗，那甚至可以只用一部手机。

VLC视频流查看

使用VLC media player可以播放网络来源视频流，查看视频流服务器的推流效果。开源的，很好用。VideoLAN是一个非盈利项目，他们还有个VLMC剪辑工具，但是目前只有源码，还没有测试版可以下载。
【VLC官网】https://www.videolan.org/
主菜单-媒体-打开网络串流。然后输入视频流的地址。
这里需要注意，输入的视频流地址不带端口号
例如推流时是:rtmp://1.14.204.238:1935/mylive/test
拉流时就是rtmp://1.14.204.238/mylive/test
这里使用了OBS推流的画面。实测延时大概在10S左右。考虑到这是自己手搓的，我觉得算不错了。而之前测试的商用4G图传，后台查看延时只有5-6S，好的时候甚至可以缩短到3-4S。
