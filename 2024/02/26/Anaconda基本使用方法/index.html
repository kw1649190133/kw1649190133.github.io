Anaconda基本使用方法

Anaconda简介

Anaconda是一个环境管理器。当运行各种项目时，电脑中可能需要同时安装多个版本的Python，使用环境管理器避免环境之间的冲突。

可以使用Prompt打开命令行，也可以用Navigator打开图形界面

图形界面

在图形界面可以简单直观地管理环境和包

将Anaconda的安装路径添加到系统变量中。控制面板→系统属性→高级→环境变量，在系统变量中找到Path，添加下面几条。（一般情况似乎不用手动添加，可能是因为重装了系统）

F:\Softwork\Anacoda
F:\Softwork\Anacoda\Scripts
F:\Softwork\Anacoda\Library\bin
F:\Softwork\Anacoda\Library\mingw-w64\bin

命令行常用指令

环境管理

创建环境
conda create -n env_name

激活环境
conda activate env_name

退出环境
deactivate env_name

克隆环境
conda create -n new_name --clone old_name

删除环境
conda remove -n old_name --all

查看当前所有环境
conda env list

查看conda信息
conda info

可以使用克隆+删除的方式，重命名环境

包管理

查看当前环境已安装的包
conda list

安装
conda install package_name

安装多个
conda install numpy scipy pandas

安装指定版本
conda install numpy=1.10

更新包
conda update 包名
可以使用conda更新自己

卸载包
conda remove 包名

使用pip也是可以的，还有换源之类的操作略过

错误处理

输入activate，进入base环境

再输入进入环境的指令
conda activate 指定环境

关于环境安装位置的补充

–ps20240729补充
现在我已经不用臃肿笨重的Anaconda了，而使用小巧便捷的miniconda。但遇到个问题，miniconda也老是把环境装到C盘去，让本就不富裕的存储空间雪上加霜。
当安装miniconda的时候，会提示给用户安装还是给所有人安装，当选择给所有人安装之后，User用户组就会失去对Miniconda文件夹的所有权，然后当创建环境的时候，环境就会自动跑到C盘去（这叫什么破事啊）。
解决办法也很简单，找到Miniconda的安装文件夹，右键——属性——安全，给User用户组附上完全控制权。
Window系统的这权限管理真是麻烦，有时候和linux一个尿性，哈哈。
