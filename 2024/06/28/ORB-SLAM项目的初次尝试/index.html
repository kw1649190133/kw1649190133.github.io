ORB_SLAM项目的初次尝试

前言

这次尝试一下ORB-SLAM3项目。据说这个项目时支持单目、双目、RGBD相机、激光雷达等多种模式的。

硬件版本：
OrangePI 5 Plus RK3588 16G
ubantu 22.04

【官方项目仓库】https://github.com/UZ-SLAMLab/ORB_SLAM3

把项目下载下来之后，尝试一下进行编译，看看需要哪些部件。可以看到主要缺少的就是DBoW2、g2o、Sophus这几个东西。还有Cmake，这个至关重要的编译工具。

6、安装Boost库

我还不知道这什么什么玩意，请教了一下度娘。
Boost是为C++语言标准库提供扩展的一些C++程序库的总称。Boost库是一个可移植、提供源代码的C++库，作为标准库的后备，是C++标准化进程的开发引擎之一，是为C++语言标准库提供扩展的一些C++程序库的总称。
这东西不是标准库，但是非常标准。
【官网】https://www.boost.org/
需要注意下载.tar.gz文件（tar.gz一般用在Linux系统上，而zip用在Windows系统上。）
解压下载的压缩包，进入文件夹内，执行安装指令。

头文件默认安装在/usr/local/include下，库文件默认安装在/usr/local/lib下。

7、安装PANGOLIN

Pangolin是OpenGL的显示库，据说简单小巧，用起来很容易。
Pangolin的依赖一般有：Glew，CMake，Boost，Python
到github下载压缩包。git clone不好用。
仓库地址：https://github.com/stevenlovegrove/Pangolin/releases

git clone报错

一般而言安装应用总会遇到各种问题，但我从来没想过会在下载的时候就撞南墙。这玩意根本下不了，用git clone会无限报错，在github页面也无法下载完全，压缩包只有一丁点大明显不正常。

报错：curl 16 Error in the HTTP2 framing layer
解决办法：git config –global http.version HTTP/1.1 #强制使用1.1，也可以使用同样的办法切换回去。

报错：error: RPC failed; curl 18 transfer closed with outstanding read data remaining
参考：https://blog.csdn.net/fan1865221/article/details/105478673
解决办法：

【……算了别用git clone了，直接去页面点击链接下载比较好。】

PANGOLIN运行测试

进入编译后的目录下，找到样例程序：Pangolin/build/examples/HelloPangolin
编译成功的话，这里是会生成可执行文件的，使用./HelloPangolin运行可执行文件。可以得到一个三色方块。

8、安装ORB-SLAM3

在前面的各种关联程序都安装完成之后，开始安装主程序。如果遇到问题，就查看一下报错的是哪个部分，然后去单独处理。

前期准备：修改测试样例的设置

参考了KSilver的教程：https://www.bilibili.com/read/cv24690326/
这里提到了修改样例程序的设置，就免得后面再重新编译一次。
进入Examples→Monocular，打开mono_euroc.cc文件，把第83行的false改成true。（这一步是为了程序运行时能显示可视化窗口）

编译安装

最后阶段，开始编译主程序

其实打开build.sh文件，里面是一堆指令的集合。可以把make -j改成make -j4，这样可以避免-j调用的核心数量过多时内存负担过大。不过我的16G的RK3588应该没有这种问题。

编译成功之后，进度条会显示100%，并且没有任何报错。

错误处理：编译失败

错误描述：进程到built target g2o的时候总是会卡住，然后退出。
解决办法：打开CMakeLists.txt，在project(ORB_SLAM3)的后面加上set(CMAKE_CXX_STANDARD 14)。

修改之后编译没有再遇到问题。

运行样例程序

这条指令可以分成6部分：

  1. ./Examples/Monocular/mono_euroc 可执行文件

  2. ./Vocabulary/ORBvoc.txt 用于回环检测的词袋

  3. ./Examples/Monocular/EuRoC.yaml 参数配置文件

  4. ./datasets/MH01 数据集

  5. ./Examples/Monocular/EuRoC_TimeStamps/MH01.txt 时间戳配置

  6. dataset-MH01_mono 轨迹文件的保存名称，会生成在ORB-SLAM3文件夹里

报错

报错内容：error while loading shared libraries: libpango_windowing.so: cannot open shared object file
说的是无法连接到这个库文件
解决办法：sudo ldconfig重新加载配置文件。
