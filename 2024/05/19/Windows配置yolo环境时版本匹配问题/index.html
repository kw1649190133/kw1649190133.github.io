Windows配置yolo环境时版本匹配问题

问题说明

错误描述

报错截图：

测试torch和CUDA也依然是可用性检查不通过，但是Torch和CUDA都是正常安装了的，使用nvcc --version是可以正常看到结果的。

错误缘由

在某些时候，因为环境和CUDA安装的先后顺序问题，可能会导致自动安装的ultralytics环境会安装到版本不匹配的Torch和Torchvision，然后运行的时候就卡住了。

解决办法

参考文章：
https://blog.csdn.net/qq_61776376/article/details/136778210
需要先进入创建好的Conda环境。
首先删除旧包，反正用不了，旧的不去新的不来。

在训练环境中输入查看debug


这里会显示出支持的python版本（别惦记那破3.8了，有条件还是上11或12吧。）

去网上手动下载Torch和Torchvision
https://download.pytorch.org/whl/torch_stable.html

在纷杂的文件中找到自己对应的版本。这里使用一下原文的截图

安装刚才下载的对应的轮子。对于熟悉pip的人不用说了。要是不知道的话，就吧文件名改成绝对路径吧。

pip install 文件名.whl

最后成功跑通。

兼容性测试

在第一次部署环境的那篇文档中也有，这里重复一遍

cudnn好像不用专门去搞了，好像装好CUDA工具之后，cudnn就自己装好了。可能是已经内置了？

又遇到问题了

20240730 补充记录

问题描述：之前给笔记本装过环境的，但是为了从Anaconda3换成miniconda，我又把环境删了。现在不得不再次重装环境，我以为不会有什么意外了，结果意外还是发生了。

问题分析

很显然，程序报错的原因是在torch文件夹内没找到运行库文件fbgemm.dll，我顺着路径去找了一下，确实没有。
这个东西实际上是VC运行库里的东西，我看到网上有人说通过安装VC_redist就可以解决，但尝试了一下并没有用。

最终解决办法：
去TMD垃圾pip，改成用conda就没问题了。

算了，pip捡起来擦擦还能用。

再次安装torch和torchvison

现在运行库没有问题了，回到这篇文章的问题本身，只要torch和torchvison版本匹配就可以了。这里我试了一下使用的是torch2.2.0
（后来发现不行，还得是2.2.1）

然后我又仔细看了一下，原来是torchvison版本没对上，根本就没安装上。

更换为torchvision=0.17.0

成功import

再次尝试import，成功；监测CUDA匹配，成功！这下搞定了。

查询依赖库

另外，在尝试解决问题的过程中找到了一个查询文件依赖的软件，虽然这次没有用上，但感觉以后可能会用到，特此记录一下。
https://github.com/lucasg/Dependencies

大起大落

高兴早了，怎么还有问题

不是，这个问题怎么还存在啊，自检不是都通过了吗。只能先试试重启了，实在不行再换一下torch版本。

再次重装torch

最后更换为了torch 2.2.1和torchvision 1.7.1，后面这个.1的颗粒度对齐非常重要，更换完之后就成功了。只有这个版本能用，真是服了。看来以后再重装环境，认准这个版本就行了。实测就是2.2.0的torch有问题，垃圾玩意。
