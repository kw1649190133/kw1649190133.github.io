Git与Github

前言

git本身就是一个进行版本管理的工具，它的作用在于：当我们对文档和代码的修改进行备份时，不再需要备份整个文件，而是只需要记录改动的部分，这样就极大的减少了冗余，降低了信息复杂度，让迭代更新变得更加简洁高效。一般通过命令行来就比较方便。它也有简单的图形界面，安装完git之后，选择git gui启动图像界面。
而github是与git衔接的网络仓库，二者并不是一个东西，只是互为配套工具。
我们比较熟悉的git clone 和git push ，就是从云端下载东西、推送仓库到云端。

GitHub 核心模块功能解析

  1. Issues（议题）：用于追踪项目中的问题、建议或任务，支持标签分类、分配负责人和状态标记，是团队沟通与需求管理的核心工具。
  2. Pull Requests（拉取请求）：开发者在分支上完成代码变更后，通过 PR 提交合并请求，便于代码审查、讨论与质量把控，是协作开发的关键环节。
  3. Agents（智能代理）：依托 AI 模型自动执行仓库维护任务，如自动为 Issue 添加标签、生成依赖更新 PR、识别语法错误并提供修复建议，提升开发效率。
  4. Actions（自动化工作流）：通过 YAML 配置文件定义 CI/CD 流程，实现代码提交后自动测试、构建、部署等操作，保障代码质量与发布效率。
  5. Projects（项目管理看板）：提供可视化看板界面，可将 Issues 和 PR 拖拽至不同列（如“待办”、“进行中”、“已完成”），直观跟踪项目进度与任务分配。

文件与文件夹描述的修改方式

GitHub 上的文件和文件夹本身没有独立的“描述字段”，其说明信息通常通过以下方式体现和修改：

  1. README.md 文件：这是项目根目录下的标准说明文件，用于介绍项目用途、安装步骤、使用方法等。修改它即可更新项目整体描述。操作路径：进入仓库 → 点击 README.md → 点击右上角铅笔图标编辑 → 填写提交信息后保存。
  2. 提交信息（Commit Message）：每次提交代码时填写的说明文字，会显示在文件列表中作为该次变更的简要描述。例如图中“v1.2 贴近测量面板重设计 + VLM兼容 + UI优化”即为提交信息。修改方式：需重新提交一次包含相同文件变更的 commit，并在提交信息中更新描述。
  3. 文件内注释或文档：对于代码文件或配置文件，可在文件头部添加注释说明其用途；对于文档类文件（如 .md、.txt），直接编辑内容即可。操作路径：点击目标文件 → 点击右上角铅笔图标 → 编辑内容 → 提交更改。

图形化界面Git Gui

进去的界面很简洁，就三个功能，创建新的git目录、从网上已有的仓库、打开git目录。gitgui让不熟悉命令行的人也能用git，但不熟悉命令行的人一般也不需要git，他们写文档也不会用MD，而是word。所以GUI的作用因人而异，也许有的人就是需要呢。

本地项目配置

.gitignore文件一般位于仓库根目录下，标记出不需要进行版本管理和提交的文件路径。规则随项目代码一起提交，对所有克隆该仓库的开发者生效。子目录中的 .gitignore仅对该目录及其下级生效 。

位置‌：通常在用户主目录下，如 ~/.gitignore_global 或 $XDG_CONFIG_HOME/git/ignore。
‌作用‌：对所有 Git 仓库生效（如忽略 .DS_Store、IDE 配置文件等）。

github设置管理

创建的github仓库分为公有和私有两大类，公有就是基本公开，谁都可以访问和下载，私有就是只有仓库所有人和共同开发者才能下载。
而如何认定权限，以前是通过账号和密码，但为了避免密码的频繁输入造成数据泄露，后来github又强推了token模式，通过令牌进行仓库访问，降低密码泄露风险

仓库转为私有

在仓库页面找到设置——settings——General——DangerZone（危险选项，最底下）——change repository visiblility，可以改变仓库属性
私有化之后，路人就看不到这个仓库了，如果需要与别人协作，大致有两种方式。

  1. 添加对方GitHub账号为协作者，这样更方便也更规范，对方只需要使用自己的密钥体系。
  2. 如果对方没有GitHub账号，也懒得创建，可以自己做一个访问密钥，交给对方。

为共同开发者创建访问密钥

在右上角选择账号，setting——Developer Setting——Personal access tokens——fine-grained tokens，创建访问令牌。

  1. 需要设置令牌名称、描述、时限、
  2. 在reposition access设置令牌对应的仓库
  3. 默认的新建token只有读取权限，需要修改设置参数才有写入权限。permissisons框中是空的，需要点add premissions，搜索找到Contents ，修改Access为 read and write，就能赋予读写权限了。
  4. 点击创建密钥，然后复制保存

创建成功后，对方的访问方式：

git clone https://USERNAME:TOKEN@github.com/你的用户名/仓库名.git
# 或者
git clone https://github.com/你的用户名/仓库名.git
# 登录界面切换token登录，粘贴 token

如果想在GitHub网站上看到私有化仓库，就需要添加对方为协作者（Collaborator），这也是更规范的协作方式，token只能支持git操作，无法在云端查看代码。
