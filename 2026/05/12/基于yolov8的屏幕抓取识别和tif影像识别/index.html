基于yolov8的屏幕抓取识别和tif影像识别

前言

最近需要做海滩上管线变化的AI识别，除了比较常用的照片识别、位置。
大概2月份就一直在做了，只是一直没有闲下来进行整理。

环境

环境使用的是之前就安装好的yolo环境，这里简单总结一下安装流程

  1. 安装miniconda，用于环境管理

  2. 在miniconda中创建合适的python环境

  3. 获取和安装 ultralytics 环境

  4. 检查显卡版本，安装CUDA和cuDNN加速计算
    妮维雅开发者官网(https://developer.nvidia.com/)
    下载CUDA并安装

  5. 测试Torch和CUDA兼容性，完成匹配。
    参考资料：
    pytorch官网地址：https://pytorch.org/get-started/locally/
    其他下载站：https://download.pytorch.org/whl/cu116

    数据标注

    做标注的软件很多，大部分导出是json格式，而yolo需要的是txt。
    至少需要一个训练集和一个验证集，训练集中样本放多一点。
    目录结构：
    train
    ┗ image
    ┗ lab
    val
    ┗ image
    ┗ lab
    目录结构仅作为参考，具体的结构可以通过yaml定义。
    因为这次做的是图像分割的模型，所以标签是一个封闭多边形，第一个数字表示样本类型，后面每两个数字表示一组图像坐标。

影像识别

这里的原理其实就是将巨大的二维影像切割成多个小块，每个小块单独检测，最后再将结果拼接到一起。同时，要注意地理坐标和投影的相关问题。影像要使用带有投影坐标系的，这样程序就可以读取影像的坐标，生成矢量时进行赋值。

模块依赖

⚠️ 不建议直接 pip install geopandas！
因为 geopandas 依赖 GDAL、PROJ、fiona、shapely 等复杂 C/C++ 库，在 Windows 或 macOS 上用 pip 容易失败。rasterio同理。
