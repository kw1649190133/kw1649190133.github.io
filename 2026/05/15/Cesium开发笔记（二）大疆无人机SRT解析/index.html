Cesium开发笔记（二）大疆无人机SRT解析

前言

最近有空进行一下开发。之前一直使用别人的商业无人机平台，但是只有试用权限，并且最近权限到期了，还没法续，为了日后的数据和功能展示考虑，以及实际的项目需要，长久来看还是要做自己的平台。但是开发是一个非常非常复杂的过程，尤其是只有我一个人的情况下，所以目标也不能定得太高。
首先要解决一下历史数据展示的问题，目前积累了很多无人机素材，但只是观看视频和照片或者查看三维模型显然不够直观，并且后续还要考虑和目标识别功能结合需要，所以首先考虑能不能解析历史的飞行记录，有了坐标和姿态之类的信息，就可以将数据还原到三维场景中。

大疆无人机SRT解析

SRT文件提取

大疆的无人机在拍摄照片和视频时，会同步记录拍摄的时间、坐标、高度、镜头参数等，通过明文的方式记录为一个srt字幕文件，通过解析SRT文件，就可以大致还原飞机当时的位置和姿态。
不同机型还有一些差别，像nimi3的srt字幕文件的单独的，并且信息比较少，尤其是没有云台信息。而M4E在拍照片时会记录MRK文件，在录视频时就只会保留一个视频文件，具体的信息需要使用工具从视频中提取。

MINI3pro文件解析

参考SRT字幕文件格式

iso : 110 (感光度)
表示相机感光元件对光线的敏感程度。数值越低（如100），画质越纯净；数值越高（如800以上），画面可能噪点增多，但能在暗光下获得更亮的画面。
shutter : 1/5000.0 (快门速度)
表示相机快门的开启时间，这里是 1/5000 秒。这是一个非常高的快门速度，通常用于在光线极强的环境下拍摄，或者凝固高速运动的物体。
fnum : 170 (光圈值)
代表镜头的光圈大小。注意：这里的 170 并非我们常说的 f/2.8、f/8 这样的标准光圈值。它很可能是大疆内部用于记录光圈叶片位置的一个原始数据或特定编码，而非直接的光圈F数。
ev : 0 (曝光补偿)
表示你对相机自动测光结果的亮度调整。0 代表没有进行任何增亮或减暗的补偿，完全信任相机的自动曝光。
ct : 4791 (色温)
代表白平衡的色温值，单位是开尔文（K）。4791K 是一个接近中性白的色温，略低于正午日光（约5500K），画面会显得比较真实自然。
color_md : default (色彩模式)
表示拍摄时使用的色彩风格。default 指的是普通或标准模式，相机会直出色彩鲜艳、对比度正常的画面。
focal_len : 240 (焦距)
表示当前镜头的等效焦距。这里的 240 很可能指等效于全画幅相机的 24mm 广角端，这是大多数消费级无人机主摄的标准广角视角。
dzoom_ratio: 24833 (数码变焦比例)
表示当前的数码变焦倍率。这个数值同样可能是内部编码，需要换算。例如，如果基准值是10000代表1倍，那么 24833 大约表示进行了 2.48 倍的数码变焦。
🛰️ 飞行与姿态参数
这些参数记录了无人机在空间中的位置和角度。
latitude: 20.106396 (纬度) & longitude: 110.625347 (经度)
这是无人机拍摄该帧画面时的精确 GPS 地理坐标。根据这个坐标，可以定位到拍摄地点位于中国海南省文昌市附近海域。
rel_alt: 40.200 (相对高度)
表示无人机相对于起飞点的垂直高度。这里显示无人机在当时距离起飞点有 40.2米 高。
abs_alt: 17.728 (绝对高度)
表示无人机相对于海平面的海拔高度。这里的海拔是 17.728米。
注意：通过 相对高度(40.2m) 和 绝对高度(17.728m) 可以反推出，你的起飞点海拔约为 -22.5米 (17.728 - 40.2)。这说明你可能是在一个比海平面低的地方起飞的，或者气压计受到环境影响导致读数有偏差。
📝 其他信息
SrtCnt : 3: 这是 SRT 文件的内部序号，代表这是第3条记录。
DiffTime : 34ms: 表示与上一条记录的时间间隔为34毫秒。
delta:0: 通常是某个参数变化的差值，这里为0，表示相关参数没有变化。

M4E文件解析

从视频导出的srt文件：

解析
相机参数：

  • iso: 100: 感光度为100。
  • shutter: 1/1250.0: 快门速度为1/1250秒。
  • fnum: 2.8: 光圈值为 f/2.8。这里的数值是标准化的，可以直接理解。
  • ev: 0: 曝光补偿为0。
  • color_md: default: 色彩模式为标准。
  • ae_meter_md: 1: 测光模式。1 通常代表中央重点平均测光。
  • focal_len: 24.00: 焦距为24mm（等效全画幅）。
  • dzoom_ratio: 1.00: 数码变焦倍率为1.00倍，即没有进行数码变焦。

飞机参数：

  • latitude: 20.080980: 纬度。
  • longitude: 110.374042: 经度。
  • rel_alt: -0.000: 相对高度。这是相对于起飞点的高度。-0.000 表明这一帧画面是在地面拍摄的。
  • abs_alt: -6.363: 绝对高度。这是相对于海平面的海拔高度。负值表示起飞点的海拔低于海平面。

云台参数：

  • gb_yaw: -31.9: 云台偏航角 (Yaw)。表示云台在水平方向上向左（逆时针）旋转了约 32 度。
  • gb_pitch: 0.0: 云台俯仰角 (Pitch)。表示云台镜头处于水平状态，没有向上或向下倾斜。
  • gb_roll: 0.0: 云台横滚角 (Roll)。表示云台没有发生左右倾斜，保持水平。这个一般一直都是0

其他高级参数：

  • dehaze_level: 0: 去雾等级。当前设置为0，表示未开启去雾功能。
  • dehaze_mode: 0: 去雾模式。当前设置为0，可能是自动或标准模式。

很明显，主打行业应用的M4E的信息会比老款消费级的小飞机信息更多。主要的差别就是在姿态信息、去雾参数上，还有云台角度。M4E应该是有云台信息的，一开始我也感觉
好像没有云台信息，但经过与别人的数据对比猜测，gb应该就是指的”gimbal_attitude”，但是根据使用无人机的经验，云台一般是默认朝前的，只有俯仰可以从遥控器上进行调整，横滚一般也不会发生，所以这个偏航角yaw应该也能当作是飞机的方向？

需要注意的是，航线记录中的yaw方向与平台上加载的glb模型的方向可能不一样，将模型显示在地图上的时候可能还需要做一个旋转，或者干脆在Yaw数值的处理上直接-90。

Json格式转换与Cesium开发

解析了文件之后，下一步就是考虑如何将航线显示在三维平台下。使用浏览器启动的网页平台，显然是不方便读取srt文件的，需要将srt转换成json。直接做一个浏览器上的json转换不是不行，但目前开发能力有限，还是先简单做一个py脚本就行。

让AI编辑程序的时候，考虑到数据压缩和保留的问题。

  1. 原视频是4K30帧的，每一帧都是一条srt记录，对于显示整条航线来说就有点多了，按照每一秒钟一条json记录的形式提取，可以大幅度压缩数据量。
  2. 尽可能保留所有参数，需要的时候就可以更方便的调用。

其实这样子用起来肯定是不够方便的，后续还是要做一个界面，或者改一下代码放到网页端项目里去。

JSON验证

json文件可以看作是一个多维数组，它对内容几乎没有限制，但是对格式有严格要求，少一个符号都不行。
JSON检验的工具在网上有很多，转换过程一般也不会有这种低级错误。
JSON校验：https://jsonlint.com/
格式化与压缩工具：https://www.bejson.com/
