OrangePi5Max安装server系统

烧录服务器端系统

  1. 按住MaskROM按键并接通电源，进入MaskROM模式。成功后软件会提示发现设备。

  2. 在左边对话框加载官方EMMC烧录模板，选择配置文件路径、镜像路径、勾选强制地址写入，执行烧录

    ⚠️ 注意：把镜像的名称改短一点，否则烧录程序容易报错。

    连接开发板

    我只有usb键盘和显示器，而开发板是自带wifi和蓝牙模块的，所以通过直接输入指令，理论上就能连上wifi。虽然安装的是server端，但接上屏幕也可以直接显示命令行界面。

    联网

    在命令行界面通过指令进行控制

ssh连接

前面获取到了开发板的ip地址后，就可以通过IP进行访问了。连接之后可以从windows电脑复制粘贴，免得敲代码。

香橙派官方镜像的默认名称和密码都是orangepi，通过ssh orangepi@192.168.31.159（刚刚分配的IP地址），再输入密码，即可访问。

其他方式

除此之外，还可以用开发板直接用网线或者usb连接电脑。比如USB转TTL，需要购买模块。

换源

在联网之后想要进行软件包更新和下载桌面，都需要先换源，不然网速太慢，还会有一堆404。

安装桌面

PS：后来我再次安装时发现，如果直接安装xubuntu界面，不仅安装中文会出问题，网络设置也会出问题，简直麻烦不断，所以先安装GNOME再删除安装xubuntu可能是我在不知不觉中做出的正确选择。

官方界面（失败）

一开始尝试安装官方GNOME桌面，但是遇到了进不去桌面的问题，折腾了一会只能暂时放弃。
据说官方GNOME桌面主要适配x86架构，对ARM的适配度一般，RK3588 对 Wayland 支持不成熟，需要强制 GNOME 使用 Xorg（X11）。

⚠️ 注意事项

  • 不要混用不同 Ubuntu 版本源（如 jammy vs focal），否则可能破坏系统。
  • 官方 Ubuntu Desktop 是为 x86_64 优化的，ARM64 版本虽功能完整，但部分小众软件可能缺失。

错误排查

  1. 输入密码后无法进入桌面
    暂时没法解决。退回命令行界面的方法：Ctrl + Alt + F2

安装中文界面和输入

进入界面还没有结束，还有中文界面和输入法需要改，不然无法直接在香橙派上做笔记和查阅资料，非常不方便。
⚠️ 注意：不建议将系统路径也改成中文，避免一些混乱，只改界面就可以了

🌐第一步：安装中文包

🖥️ 第二步：在 XFCE 中设置中文显示（用户级）

  1. 打开 Settings（设置） → Language（语言设置）
  2. 点击左下角 “Install / Remove Languages”
    勾选 Chinese (simplified) → Apply
    （这里程序可能会弹窗自动识别到未加载的语言包，直接apply就可以了，不用自己选）
  3. 在语言列表中，将 “Chinese (China)” 拖到顶部
  4. 点击 Close，注销（Log Out）后重新登录
    ✅ 此时菜单、设置等应变为中文（部分应用可能仍为英文，属正常现象）。
