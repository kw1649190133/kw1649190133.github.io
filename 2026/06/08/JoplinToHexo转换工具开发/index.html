JoplinToHexo转换工具开发

前言

我一直保持着用hexo博客记录开发和实验过程的习惯，但是每次用joplin打字，再转到vscode中处理，最后在命令行打包更新的流程实在是有点繁琐了。一直想给自己做一套工具，但苦于没有能力，也没有时间。
但现在不一样了，大模型即将主宰一切。

需求描述

我一般使用joplin来写文档，用hexo发布，但是期间的格式转换和数据迁移过程实在是太麻烦了。使用joplin导出MD文档后，每次我还需要在hexo中进行hexo new，然后复制文档内容到posts文件夹，修改图片路径，添加标签和分类，最后再更新上传。
hexo博客的_posts文件夹中是每一篇MD文档和同名文件夹，图片、附件都放在同名文件夹中。
1、参考文档抬头部分

每次hexo n之后，tags是空的需要自己补充。categories分类会缺失，需要手动写上去。
2、MD文档中的图片需要修改路径
从Joplin导出的MD文档，文档中的图片都是这样格式的。
![b0e8e2668b5b17192718f69cf1ba1a80.png](../_resources/b0e8e2668b5b17192718f69cf1ba1a80.png)，而实际上地址的前缀../_resources/应该被删除。
3、导出路径里面是记事本名称/文档名称/，_resources文件夹里面是所有文档的图片，一次性导出多篇文档时容易图片混淆，需要根据MD文档的图片名称，精确复制到_posts文件夹对应文档的同名文件夹中。
参考导出路径
C:\app\blog\临时
参考_posts文件夹路径
C:\app\blog\备份_posts
我希望做一个python有界面的程序，由我输入一个MD文档导出地址，和hexo博客的posts文件夹地址，然后按键读取到文档列表后，再按转换自动帮我完成文档抬头部分生成，图片拷贝迁移，标签和分类完善。我可以直接在程序里修改每篇文档的标签tags、分类categories并保存。并且，程序最好能记住我上一次运行的输入路径，这样每次打开按转换按钮就可以了。

第一次生成的程序还有点路径读取的小问题，经过反馈和修复，第二次就直接成功了，还顺便做了图标美化、界面UI调整。这样我只要在Joplin中编辑好文档，直接导出并转换就能同步到hexo里了。

最终版本，基本上该有的功能都有了，体积也不大。

不过实际使用的时候还是发现了一点小问题，例如针对../_resources/的识别没有剔除特殊情况，把说明书里的内容也删除了，实际上应该优化检测逻辑，剔除特殊情况。不过这种特殊情况比较少，以后一般也不会遇到，无所谓了。
最后，用AI生成说明书。

HexoBlogTool 使用说明书

Joplin → Hexo 博客一键转换工具

一、简介

如果你和我一样，用 Joplin 写笔记、用 Hexo 发博客，那你一定经历过这套痛苦流程：

  1. Joplin 导出 MD
  2. hexo new 创建文章
  3. 复制内容到 _posts
  4. 手动改图片路径 ../_resources/xxx.png → xxx.png
  5. 手动复制图片到文章同名文件夹
  6. 手动补 tags、categories
  7. 最后 hexo g -d 发布

HexoBlogTool 就是来解决这个问题的。填好两个路径，点一下按钮，上面的第 2~6 步全部自动化完成。

二、快速开始

2.1 运行方式

直接双击 HexoBlogTool.exe 即可启动，无需安装任何依赖。

八、常见问题

Q: 读取文章后列表为空？

  • 确认 Joplin 导出格式为 MD - Markdown
  • 确认导出目录下确实有 .md 文件
  • 检查路径中是否包含 _resources 文件夹（程序会自动跳过它）

Q: Tags / Categories 怎么编辑？

  • 在文章列表中 双击 Tags 或 Categories 列的单元格
  • 输入内容后按 回车 保存，按 Esc 取消

Q: 转换后图片不显示？

  • 确认导出目录下的 _resources 文件夹存在
  • 检查日志中是否有黄色警告提示图片缺失
  • 如果是网络图片（http://），程序不会复制，保持原链接

Q: 转换后文件已存在怎么办？

  • 当前版本直接覆盖，请确保 _posts 中没有重要的同名文件
  • 建议转换前备份 _posts 目录

Q: 程序卡住了？

  • 大量文章（50+ 篇）转换时需要一些时间，请耐心等待
  • 查看底部日志区确认进度，不要强制关闭
