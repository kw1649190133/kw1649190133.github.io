通过WSL运行PX4的AirSim仿真

前言

最近需要做无人机自主飞行的仿真模拟，找了一下别人都用什么环境，发现Gazebo、AirSim、XTDoran几个选择。AirSim作为虚幻引擎的插件，可以利用游戏引擎快速实现比较好看的渲染效果，但是比较依赖Windows环境，近几年据说维护不好；而Gazebo和XTDoran在科研和比赛中使用较多，资料也比较多，在Linux环境下使用很方便。

WSL安装

启用WSL

前提：系统和CPU硬件必须支持WSL，一般要求win10以上。

  1. 重启计算机，F2进入BIOS系统（不同品牌有区别），一般会在CPU设置里，找到并启动CPU虚拟化。
    Virtualization Technology -> Enabled

  2. 在控制面板-程序-启用或关闭windows功能中，找到适用于Linux的windows子系统并打开。如果要使用虚拟机还需要打开虚拟机平台和Hyper-V，但WSL好像不需要这些选项。（PS后面在台式机尝试了一次，好像又不开不行）

  3. 在Microsoft Store中搜索Ubuntu，选择需要的版本安装。一般常用比较稳定的20.04或者22.04，安装完成后会自动启动。

位置迁移

WSL的默认安装位置是C盘，为了避免系统盘的内存占用，最好是进行一个迁移。
wsl -l -v 查看当前wsl系统的列表和版本
wsl --export Ubuntu-22.04 D:\WSL\Ubuntu\ubuntu_backup.tar 打包迁移指定的wsl系统
wsl --unregister Ubuntu-22.04 注销原系统
wsl --import Ubuntu2204 D:\WSL\Ubuntu D:\WSL\Ubuntu\ubuntu_backup.tar --version 2 在新的位置释放压缩包
ubuntu2204 config --default-user kwindy 重置默认用户（重要）
重置默认用户后会自动进入WSL

使用指定用户启动
wsl -d Ubuntu-22.04 -u kwindy
到这里安装就结束了，但因为后续我还要使用PX4和QGC，所以后面还需要配置一下图形和端口方面的问题。

WSL下的常用操作

Linux命令行的基本操作和在虚拟机中是类似的。但由于WSL和windows实际上共享一个存储空间，在命令行中可以直接访问window硬盘。

使用cd /mnt/d 来访问，用cp /mnt/d/px4.zip ./ 可以将windows的文件复制到根目录下。

unzip px4.zip 解压缩包
解压文件的程序还得自己安sudo apt install unzip
重命名方法 mv 要重命名的文件夹路径 新的文件夹路径

配置图形环境

⚠️如果不用图像
WSL虽然基本上是支持Linux功能的，但AirSim涉及到一些图形画面，以及需要打开QGC来进行控制。WSL2 默认 不包含完整的图形驱动栈，虽然可以配合 X Server 显示窗口，但缺少必要的 OpenGL 库。

错误排查

有可能会出现无法启动的问题。
可以尝试关闭和恢复防火墙。但其实一般不会是防火墙的问题。

✅ Windows 本地 Test-NetConnection 127.0.0.1 -Port 6000 成功 → VcXsrv 确实在监听
✅ 防火墙已关闭 → 排除防火墙拦截
✅ window 本地检查监听端口 netstat -ano | findstr :6000 显示 0.0.0.0:6000 LISTENING → 监听地址正确
❌ 但 WSL2 中 xeyes 仍报错：Can’t open display: 10.255.255.254:0.0

10.255.255.254 是 /etc/resolv.conf 中的 nameserver（通过cat /etc/resolv.conf查看），但它在某些 Windows/WSL2 状态下是“虚拟网关”，并不真正响应 TCP 连接。而通过window下ipconfig找到的WSL网络适配器IP，或者通过linux中ip route show找到的via，才是真实的WSL网关。

解决办法

AirSim安装过程

简单介绍

AirSim 是一个开源的仿真平台，主要用于无人机和自动驾驶汽车的仿真测试，基于Unreal Engine开发，提供高保真的视觉渲染和物理模拟。‌它支持多种传感器模型（如摄像头、IMU、GPS）和编程语言接口（如Python、C++），适用于研究与开发场景。‌

AirSim 已停止维护，仅支持到UE4
文档：https://microsoft.github.io/AirSim/
Colosseum 支持UE5
仓库：https://github.com/CodexLabsLLC/Colosseum
文档：https://codexlabsllc.github.io/Colosseum/
另外，AirSim也有一定的局限性，好像目前已经不再继续更新了，据说导航和控制用gazebo，做强化学习用isaaclab。

前期准备

  1. 安装Epic game游戏平台。这个平台已经将游戏和虚幻引擎开发绑定在一起了。
  2. 在epic中下载虚幻引擎的4代版本
  3. 安装VisualStudio 2022。注意是2022版，安装2026等其他版本可能会导致AirSim编译失败。安装时只需要勾选C++桌面开发组件，NET framework版本需要注意兼容，以及window 10/11 SDK（一般会默认选择）。

编译AirSim

  1. 去网站下载AirSim的项目文件
    https://microsoft.github.io/AirSim/
  2. 搜索Developer Command Prompt for VS 2022，打开安装好的VisualStudio2022环境命令行，cd指令进入AirSim文件夹，执行build.cmd指令，开始编译，正常编译时间三到五分钟。

错误：MSBUILD：error MSB1009：项目文件不存在
解决：visual studio版本太高，组件不匹配，重装2022版，或者安装官方说明支持的2019。

启动进入界面

成功编译之后，双击启动AirSim\Unreal\Environments\Blocks\Blocks.sln，第一次可能需要关联visual studio

  1. 切换配置Debuggame Editor （调试）
  2. 右键Blocks设为启动项目
  3. 运行本地windows调试
  4. 成功进入虚幻引擎，再点击虚幻引擎的运行。

安装完成，AirSim内置了简易操控程序，此时可以直接插上游戏手柄，用手柄摇杆直接控制无人机，但这么做只是玩游戏，真正的无人机控制应该由PX4完成，接下来要解决的是AirSim与PX4环境的联动问题了。

使用PX4控制AimSim仿真

原理

核心原理：分布式 SITL 仿真架构
PX4 的 Software-In-The-Loop (SITL) 设计允许飞控逻辑运行在一台机器上，而物理/视觉仿真器（如 AirSim）运行在另一台机器上，两者通过 TCP/UDP 协议通信。
| 组件 | 角色 | 通信方式 | 端口 | 方向 |
|——|——|——–|——|——|
| PX4 SITL | 飞控仿真核心
（姿态控制、导航、状态估计） | TCP Server | 4560 | 被动监听，等待 AirSim 连接 |
| AirSim | 物理/传感器仿真器
（提供 IMU、GPS、气压计、视觉等数据） | TCP Client | 连接 4560 | 主动发起连接 到 PX4 |

几种启动的区别

  • none 纯逻辑启动，不输出外部图像
  • jmavsim PX4本地简单仿真，需要支持图形渲染显示
  • none_iris 连接外部飞行器，需要配置端口和等待设备响应

启动PX4和AirSim连接

通过visual studo启动虚幻引擎编辑环境，运行环境，看到等待TCP连接的提示。
确认地址配置无误，启动PX4外部连接模式。

提示realy for takeoff则连接成功。在WSL新开一个窗口，可以使用QGC控制AirSim的无人机了。

关于启动顺序有一些说法，需要先启动UE再启动PX4，并且断开之后默认是没有重连机制的。

关于流畅度

因为测试中我们的控制是通过QGC进行的，当虚幻引擎被置为后台时会默认进行降帧，减少GPU占用，让画面变卡，如有需要可以在编辑——编辑器偏好设置——通用——性能中，取消置于背景时自动减少GPU占用。

无法启动的问题

WSL有可能会遇到jmavsim都无法启动的问题，如果排除了X11图像环境安装，
尝试PING主机WSL的IP是成功的，但是每次运行都会卡在INFO [simulator] Waiting for simulator to accept connection on TCP port 4560，尝试telnet localhost 4560无法访问4560端口。

排除方法：

  • 可以尝试关闭防火墙，但一般不是防火墙原因。
  • 主要检查bashrc参数是否配置正常，是否正确写入，格式是否正确
  • X11图像是否可以使用，测试眼睛、齿轮等样例程序
  • ip连接是否畅通，端口是否开放。电脑装过其他虚拟机环境，可能让WSL无法找到正确的主机地址，参考WSL安装的说明。
  • AI的建议只能参考，不能全信，尤其是增加复杂度的地址转发等方案。

AirSim场景设置

AirSim中自带了一个Block场景，但在虚幻引擎商店中可以找到一些付费或免费的资源，区分工程类 Engine 和资源类 Project，工程类可以直接加载打开，资源类需要下载后自己添加到项目中。并且需要注意资源包的兼容性问题，资源包有些能兼容UE4，有些仅支持UE5。

Unreal Engine入门

点击右上角可以启动虚幻引擎编辑器，可以选择进入历史项目或新项目

新项目类型选择游戏，空白模板，选择蓝图模式（默认设置），修改项目存储位置。蓝图模式对不熟悉C++环境的人来说比较友好。

场景加载成功后，可以看到编辑器界面。下面是资源管理器，左边可以添加物体，右边可以查看列表和设置属性。首先大致理解场景包是如何构成的，它包含场景中各种物体的模型、表面材质包、环境背景音、天空盒等等。其中汇总描述整个场景的文件，一般是关卡（level）或者地图（map），通过过滤器找到之后双击就可以在编辑画面加载。（内存开始大量占用）

创建C++类

主菜单——文件——生成C++类
新建C++类，选择无类型。

创建成功后，会自动弹出visual studio，打开新创建的cpp文件。目前还不需要进行编辑，创建C++类主要是为了生成VS的工程文件sln，便于从VS进入虚幻引擎环境。

添加素材

在Epic Games中选择Fab，可以搜索Rura Australia，或者其他的免费素材。以前是集合在虚幻引擎页面的直接就可以插入工程，现在改了版本，单独开了一个素材商店，免费素材也需要先添加到购物车，添加到库，再添加到工程。

添加资源之后，可以在内容浏览器看到Rura Australia文件夹，里面包含了从场景到物体到材质的各种资源。
添加过滤器——关卡（level），双击或者按右键就会直接加载。

挑个好位置，把出生点拖出来放上去。保存。

如果需要将当前关卡置为项目启动默认关卡，在主菜单——编辑——项目设置中进行修改。在地图与模式中可以设置编辑器和游戏启动时的默认地图。

AirSim的导入

  1. 复制文件
    在AirSim/Unreal/Plugins，就是AirSim的插件，可以直接复制到其他的Unreal项目中

  2. 检查设置
    主菜单——编辑——插件，启用AirSim（复制后一般是默认启用的）
    主菜单——窗口——世界场景设置，游戏模式重载选择AirSimGameMode。

  3. 运行成功

已经废弃不用的方法

之前有直接修改.uproject，添加Modules和Plugins内容，再重新编译的方法，因为VS编辑器的代码有一些变化，这种直接修改工程文件的方式可能会报错，不推荐使用。只需要将AirSim的插件文件夹复制到项目里，虚幻引擎会自动识别的，只需要在引擎里设置就可以了。
⚠️这个uproject文件有json一样的严谨结构，结构有一点错误都会导致无法运行，不建议手动修改它。
